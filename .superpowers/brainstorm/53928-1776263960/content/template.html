Which template should we pull visuals from?

The layout we port into @dds/renderer will inherit this template's tri-column structure, typography, and dark aesthetic. All three are MIT-licensed and Next.js-native. Click to select.

A

Fumadocs roadmap pick · 11.3K★

What it is: Modern Next.js-native docs framework. Powers shadcn/ui docs and many modern OSS projects.

Visual vibe: Dark-first, editorial, sharp typography, subtle borders. Reads like a technical wiki. Tri-column: hierarchical left nav + content + right TOC.

3D compatibility: ★★★★★ — dark theme, low-contrast chrome, plays well behind a subtle scene.

Portability: Built on MDX + Next.js App Router. Components are unstyled primitives + Tailwind — easy to lift the shell, throw away MDX, feed our UniversalSection.

B

Tailwind Plus · Protocol premium · API-docs flagship

What it is: Paid Tailwind Plus template. The reference design for modern API documentation.

Visual vibe: Extremely polished. Frosted-glass nav, subtle radial glow behind headings, typography-first. Used as the gold standard by Stripe, Linear, Resend docs.

3D compatibility: ★★★★★ — already ships with ambient "glow" effects; a WebGL scene replaces those gradients cleanly.

Portability: Pure Tailwind + Headless UI. No MDX dependency. Highest-fidelity starting point, but paid license — we'd lift patterns, not fork the repo.

C

Nextra Vercel-maintained · 13K★

What it is: Vercel's official Next.js docs framework. Powers the Turbo, SWR, and React Flow docs.

Visual vibe: Clean, minimal, Vercel-adjacent. Light-first (dark mode fine), narrower editorial feel, less atmospheric.

3D compatibility: ★★★ — more content-dense, less "ambient" space. Scene would fight for attention.

Portability: Tightly coupled to MDX theming and its own theme system. Lifting the shell means undoing more framework assumptions.

My recommendation

Fumadocs (A). It matches the roadmap, its chrome is the least "opinionated" so our brand tokens can drive visual identity, and its dark-first defaults mean the three.js scene reads as design, not decoration. Protocol (B) is higher fidelity but the paid license makes "just clone it" awkward — we'd recreate the patterns in our own components anyway, so pulling from Fumadocs and borrowing Protocol's atmosphere is the pragmatic path.
